## Gateway Incident (CRITICAL LESSON)

### What Happened
- Gateway started flapping (dying every ~20 min with code 1000 - graceful close, no reason)
- I disabled 3 lifeline cron jobs thinking it would help reduce overhead
- This REMOVED the safety net - the lifelines auto-restart the gateway when it dies
- Gateway got into a bad state and couldn't recover
- Wyatt had to manually run: `openclaw gateway --force --auth none`
- Took ~1 hour to fix

### Root Cause
Gateway keeps dying with code 1000 (normal closure) but no reason. Unknown origin.

### Recovery Command
`openclaw gateway --force --auth none`

### What NOT to Do
- NEVER disable the lifeline cron jobs (Gateway Lifeline, Gateway Super-Lifeline, Gateway Emergency)
- They exist for exactly this situation
- Don't use `openclaw gateway stop` to recover - just use --force
